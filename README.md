# M4 Architecture: Post-Quantum Structural Data Fragmentation (PoC)

## Overview
M4 is a novel cryptographic architecture designed specifically for Data-at-Rest protection (e.g., Secure NAS, Hardware Security Modules). Unlike traditional block ciphers (such as AES) that mask data while preserving its underlying mathematical geometry, M4 physically disrupts the data's structure using Singular Value Decomposition (SVD) combined with Modulo arithmetic.

## 1. Positioning in the Post-Quantum Era (vs. NIST Standardization)
During architectural reviews, questions often arise regarding M4's relation to current NIST Post-Quantum Cryptography (PQC) standards and quantum threats (Shor's and Grover's algorithms).

* **NIST PQC Focus:** The current NIST standardization process primarily addresses asymmetric cryptography (Key Encapsulation Mechanisms like CRYSTALS-Kyber and digital signatures) to mitigate Shor's algorithm. M4 does not compete with these mechanisms; it operates in a parallel domain—symmetric structural fragmentation for Data-at-Rest.
* **Mitigating Grover's Algorithm:** Grover's algorithm theoretically halves the bit-security of symmetric ciphers (reducing AES-256 to AES-128) by accelerating brute-force attacks. However, AES relies on a "Verification Oracle"—an attacker knows when a brute-force guess is correct because the decrypted output forms a readable file structure (e.g., a valid file header). 
* **The M4 Solution (No Verification Oracle):** M4 completely eliminates this oracle. A breached M4 database yields Maximum Entropy (white noise). If a quantum computer attempts to brute-force the M4 permutation keys, it will assemble the noise into millions of mathematically valid, but semantically random files (The Mosaic Effect). The attacker cannot mathematically prove which assembly is the original data, rendering brute-force approaches inherently ineffective.

## 2. Core Mechanism: SVD + Modulo Arithmetic
The algorithm bifurcates any data matrix into two distinct components. By utilizing Modulo arithmetic, the floating-point inaccuracies typical of SVD are completely bypassed, resulting in **100% Lossless reconstruction (MSE = 0.0)**.

### The Spectral Component (The Operator)
This acts as the "skeleton" of the data, derived from the U, Sigma, and V^T matrices of the SVD. It contains floating-point values that describe the general geometric structure of the original matrix.

### The Residual (The Ash)
This is the exact integer difference between the original data and the approximated SVD skeleton, wrapped in a Modulo ring. Visually and statistically, it represents pure white noise.

## 3. Addressing Technical Specifications

### A. Deterministic Reconstruction of the Operator
A common technical question is how the floating-point values of the Spectral Component (Operator) can be reconstructed perfectly without precision loss.
* **Mechanism:** During the bifurcation phase, the floating-point matrices are NOT compressed, rounded, or mathematically altered. Instead, they are subjected to pure spatial dislocation. 
* **Reversibility:** The scrambling is driven by a deterministic Chaos Generator (SHA-256 seeded with a Master Key + unique session Nonce/IV). The exact sequence of chaotic indices is perfectly regenerated by the legitimate receiver. Using inverse permutation properties (e.g., `argsort`), every floating-point value is placed back into its exact original coordinate.

### B. Computational Complexity and Bit-Depth Scaling
SVD is a computationally heavy operation with a complexity of O(N^3). 
* **Block Processing:** To maintain performance, M4 does not compute SVD on massive unified files. Data is segmented into fixed-size blocks (e.g., 64x64 matrices), which are processed independently.
* **Scaling Beyond 8-bit:** The current PoC utilizes Modulo 256 for standard 8-bit byte processing. However, the architecture is mathematically agnostic to data depth. For higher precision requirements (e.g., 16-bit telemetry or 32-bit financial transactions), the modulo ring simply scales to Modulo 65536 or 2^32, leaving the core algorithm entirely unchanged.
* **Hardware Acceleration:** Software implementations (Python/C++) will always face SVD bottlenecks. Therefore, the ultimate deployment target for M4 is an FPGA/ASIC implementation, where matrix operations and block processing can be heavily parallelized on the hardware level.

## 4. Deployment: Split-Channel Architecture (OOB)
Because M4 physically separates data into a core skeleton and a residual noise matrix, it natively supports Out-of-Band (OOB) transmission and Physical Data Sharding.
* **Single-Channel:** The Operator and Residual can be packaged together for standard secure transmission.
* **Split-Channel (Recommended):** The Residual (bulk data) can be transmitted via standard broadband or stored on generic hard drives. The Operator (the key skeleton) can be transmitted via a highly secure separate channel (e.g., dedicated VPN, 5G) and stored inside a Hardware Security Module (HSM). An attacker must compromise two entirely separate physical infrastructures simultaneously.
