# M4 Architecture: Post-Quantum Structural Data Fragmentation (PoC)

## Overview
M4 is a novel cryptographic architecture designed specifically as the foundational core for Post-Quantum Data-at-Rest protection. Unlike traditional block ciphers (such as AES) that mask data while preserving its underlying mathematical geometry, M4 physically disrupts the data's structure using Singular Value Decomposition (SVD) combined with Modulo arithmetic.

**Target Market & Positioning:** M4 is explicitly *not* designed as a B2C protocol for real-time mass communication (e.g., standard messengers or web traffic). It is a highly specialized **B2B / B2G** architecture engineered for Extreme-Assurance Cold Storage, Hardware Security Modules (HSMs), and Zero-Trust environments where data integrity and physical separation are paramount.

## 1. Positioning in the Post-Quantum Era
During architectural reviews, questions often arise regarding M4's relation to current NIST Post-Quantum Cryptography (PQC) standards and quantum threats (Shor's and Grover's algorithms). M4 serves as the ultimate structural fail-safe in a post-quantum world.

* **NIST PQC Focus:** The current NIST standardization primarily addresses asymmetric cryptography (like CRYSTALS-Kyber) to mitigate Shor's algorithm. M4 operates in a parallel domain—symmetric structural fragmentation for Data-at-Rest.
* **Mitigating Grover's Algorithm:** Grover's algorithm theoretically halves the bit-security of symmetric ciphers by accelerating brute-force attacks. However, AES relies on a "Verification Oracle"—an attacker knows when a brute-force guess is correct because the decrypted output forms a readable file structure.
* **The M4 Solution (No Verification Oracle):** M4 completely eliminates this oracle. A breached M4 database yields Maximum Entropy (white noise). If a quantum computer attempts to brute-force the M4 permutation keys, it will assemble the noise into millions of mathematically valid, but semantically random files (The Mosaic Effect). The attacker cannot mathematically prove which assembly is the original data, rendering brute-force inherently ineffective.

## 2. Core Mechanism: SVD + Modulo Arithmetic
The algorithm bifurcates any data matrix into two distinct components. By utilizing Modulo arithmetic, the floating-point inaccuracies typical of SVD are completely bypassed, resulting in **100% Lossless reconstruction (MSE = 0.0)**.

* **The Spectral Component (The Operator / Key):** This acts as the "skeleton" of the data, derived from the $U, \Sigma$, and $V^T$ matrices of the SVD. It contains floating-point values that describe the general geometric structure of the original matrix.
* **The Residual (The Ash / Bulk Data):** This is the exact integer difference between the original data and the approximated SVD skeleton, wrapped in a Modulo ring. Visually and statistically, it represents pure white noise.

## 3. Addressing Technical Specifications & Compute Power

### A. Deterministic Reconstruction
* **Mechanism:** The floating-point matrices are NOT compressed or rounded. They are subjected to pure spatial dislocation driven by a deterministic Chaos Generator (SHA-256 seeded with a Master Key + unique session IV/Nonce to prevent Known-Plaintext Attacks).
* **Reversibility:** The exact sequence of chaotic indices is perfectly regenerated by the legitimate receiver, placing every value back into its exact original coordinate.

### B. Computational Complexity and Hardware Estimation
SVD is a computationally heavy operation with a mathematical complexity of $O(N^3)$.
* **Compute Estimation:** In this Python PoC, a modern standard CPU can process a 256x256 data block in milliseconds. However, scaling this to Gigabytes of data using pure software processing is practically unfeasible due to the $O(N^3)$ bottleneck.
* **Hardware Acceleration (The NAS Vision):** M4 is intentionally designed to be offloaded to hardware. The ultimate deployment target is a dedicated Hardware Appliance (Secure NAS) utilizing FPGA, ASIC, or GPU acceleration, where heavy matrix operations are parallelized.
* **Bit-Depth Scaling:** The architecture is mathematically agnostic to data depth. Whether using Modulo 256 (8-bit) or Modulo 65536 (16-bit), the core algorithm remains unchanged.

## 4. Execution Pipeline: Reintegration & Seed Mechanics
To understand how the data is reassembled at the code level without data loss, the reintegration pipeline executes the following deterministic steps:

1. **Seed & IV Retrieval:** The system retrieves the unique, plaintext `File Nonce` (IV) stored alongside the encrypted data. The user inputs the Master Password. The core concatenates these to generate the exact deterministic SHA-256 Chaos Sequence.
2. **Descrambling (Inverse Permutation):** Using the mathematical properties of inverse permutations (via `np.argsort`), the chaotic sequence perfectly maps the scattered coordinates of both the Residual and the Spectral Operator back to their original matrix structures.
3. **Skeleton Matrix Recreation:** The descrambled floating-point matrices of the Operator are multiplied ($U \times \Sigma \times V^T$) to mathematically rebuild the approximate structural skeleton of the original data.
4. **Modulo Reintegration:** The final, lossless reconstruction is achieved by simply adding the integer Residual matrix back to the floating-point approximation matrix within the strict boundaries of the Modulo ring: `(Approximation + Residual) % 256`. The exact original bytes are restored.

## 5. Key Advantages & Primary Use Cases
M4's unique structural bifurcation offers distinct advantages over traditional encryption:

* **Split-Knowledge Architecture (Out-of-Band):** The Operator and the Residual can be physically separated. The Residual (heavy noise) stays on a generic server, while the Operator (the structural key) is stored offline on an executive's HSM or transmitted via a separate 5G/VPN channel. An attacker must compromise two physically independent infrastructures simultaneously.

**Primary Use Cases:**
* **Zero-Trust Cold Storage:** Cryptographic exchange seed-phrase vaults, where calculation speed is irrelevant, but absolute mathematical security against future quantum threats is required.
* **B2G & Military Archives:** Securing classified state data, drone telemetry logs, or strategic blueprints for long-term (10-50 years) storage.
* **Genomic & Medical Data Banks:** Ensuring patient DNA sequences cannot be reconstructed even if the main hospital database is fully breached.
